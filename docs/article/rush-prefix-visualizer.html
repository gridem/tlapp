Rush State Walkthrough

A compact view of the actual stored per-node Rush state. Each step shows for each node the stored committed prefix and local core. Derived quantities such as majority prefixes and support(prefix) are computed inside mergeState(...); they are not represented here as standalone fields because they are not actual stored state. The global proposed set and in-flight stateMsgs queue are omitted here to keep the walkthrough compact.
